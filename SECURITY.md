# Security Policy

## Supported Versions

This project is a small static template and does not currently maintain versioned security support branches. Security fixes, when accepted, are applied on the default branch.

## Reporting a Vulnerability

Please do not open a public issue for a suspected security problem.

Instead:

1. Contact the maintainer privately if contact details are available.
2. If private contact is not available, open a minimal issue requesting a private reporting path without disclosing exploit details.

When reporting, include:

- A clear description of the issue
- Steps to reproduce it
- The affected file or feature
- Any suggested remediation, if known

## Scope Notes

Because this is a static GitHub Pages template, the main realistic security areas are:

- Unsafe handling of attendee-provided data
- Third-party CDN dependency risk
- Misconfiguration of published event data
